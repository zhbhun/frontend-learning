# 模型交付检查表

用于接收外部模型、资源市场模型或设计同事交付物。每个模型进项目之前至少填一次；如果压缩、换贴图或重新导出，也要更新记录。

## 基本信息

- [ ] 资产名称：
- [ ] 版本 / 日期：
- [ ] 来源链接或交付人：
- [ ] 原始格式：
- [ ] 运行时目标格式：
- [ ] 目标页面 / 使用场景：

## 授权

- [ ] 允许当前项目使用。
- [ ] 允许商业使用。
- [ ] 允许修改、压缩和格式转换。
- [ ] 允许把优化后的模型部署到目标域名或 CDN。
- [ ] 已记录署名要求、禁止事项和授权凭据位置。

## 文件和依赖

- [ ] 主文件存在，例如 `.glb` 或 `.gltf`。
- [ ] 如果是 `.gltf`，同级 `.bin`、贴图和其他外部资源都齐全。
- [ ] 文件名不含空格、中文、临时后缀或不稳定版本号。
- [ ] 目录结构已按项目约定整理。
- [ ] 已记录文件体积、贴图数量和最大贴图尺寸。

## 尺寸和坐标

- [ ] 模型真实尺寸已确认。
- [ ] 根节点位置、旋转和缩放符合项目约定。
- [ ] 原点适合当前用途，例如产品中心或底部中心。
- [ ] 模型正面方向已确认。
- [ ] `Box3` 取到的世界尺寸符合预期。

## 几何和层级

- [ ] 顶点数和三角面数在目标设备预算内。
- [ ] 需要交互、隐藏、换材质或动画绑定的节点名称稳定。
- [ ] 不需要的空节点、重复 mesh 和隐藏对象已清理。
- [ ] 法线方向正确，没有明显破面或黑面。
- [ ] 是否允许合并 mesh 已记录。

## 材质和贴图

- [ ] 材质在 glTF PBR 结构中可表达。
- [ ] 颜色贴图和数据贴图已区分。
- [ ] normal、roughness、metalness、alpha、AO 等贴图用途已记录。
- [ ] 透明材质数量和排序风险已检查。
- [ ] 贴图尺寸符合目标设备预算。

## 动画和扩展

- [ ] 动画片段名称、时长和循环方式已记录。
- [ ] 骨骼、蒙皮、morph target 是否存在已记录。
- [ ] `extensionsUsed` 已检查。
- [ ] Draco、Meshopt、KTX2、WebP、AVIF 等运行时依赖已记录。

## 验证结果

- [ ] glTF Validator 无 error。
- [ ] Khronos glTF Sample Viewer 可正常预览。
- [ ] 项目查看器可正常加载、取景、控制和释放。
- [ ] Network 面板中主文件、外部资源和解码器请求正常。
- [ ] 优化前后视觉差异可接受。

- [ ] 可进入项目。
- [ ] 需要重新导出。
- [ ] 需要重新压缩。
- [ ] 需要美术或建模侧修复。
