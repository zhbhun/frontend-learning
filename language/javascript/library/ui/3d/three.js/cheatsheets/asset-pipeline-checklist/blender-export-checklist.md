# Blender 导出检查表

用于把 Blender 场景整理成 three.js 可加载的 glTF / GLB。导出前做结构整理，导出后再用 Validator、标准查看器和项目查看器确认结果。

## 场景整理

- [ ] 删除不需要的隐藏对象、测试灯光、临时相机和草稿节点。
- [ ] 保留需要交互或动画绑定的节点层级。
- [ ] 给关键节点、材质和动画片段命名。
- [ ] 记录哪些节点不能被后续优化合并或删除。

## 尺寸和变换

- [ ] 确认单位和真实尺寸。
- [ ] 检查模型正面方向。
- [ ] 根据用途设置原点，例如中心或底部中心。
- [ ] 对静态模型应用必要的旋转和缩放。
- [ ] 对带动画或骨骼的模型，先确认应用变换不会破坏绑定。

## 几何

- [ ] 法线方向正确。
- [ ] 没有明显重复面、破面或非预期的极高细分。
- [ ] UV 展开可用于当前贴图。
- [ ] 需要保留的顶点色、morph target 或骨骼权重已确认。

## 材质和贴图

- [ ] 使用 glTF 可表达的 PBR 材质结构。
- [ ] 复杂程序节点已烘焙成图片贴图，或明确接受导出损失。
- [ ] 颜色贴图、normal、roughness、metalness、alpha、AO 的用途清楚。
- [ ] 贴图文件存在，路径稳定，尺寸合理。
- [ ] 透明材质数量和叠放顺序已检查。

## 动画

- [ ] 只导出需要的动作。
- [ ] 动作名称稳定。
- [ ] 循环段、帧率和时长符合运行时预期。
- [ ] 导出后在查看器里检查骨骼、morph target 和动作切换。

## 导出选择

- [ ] 部署优先选择 `.glb`。
- [ ] 需要调试结构、外置贴图或 CDN 分发时选择 `.gltf + .bin + images`。
- [ ] 确认是否导出相机、灯光和动画。
- [ ] 确认贴图是否嵌入、复制或引用外部路径。
- [ ] 导出文件名带语义版本或构建版本。

## 导出后验证

- [ ] glTF Validator 无 error。
- [ ] Khronos glTF Sample Viewer 能正常显示材质、动画和尺寸。
- [ ] three.js 项目查看器能正常加载和自动取景。
- [ ] 如果后续压缩，压缩前结果已归档。
- [ ] 导出设置、Blender 版本和文件路径已记录。
