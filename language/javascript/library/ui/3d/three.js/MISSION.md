# 使命：three.js

## 为什么学习
系统学习 three.js，从零开始快速上手浏览器 3D 开发，并形成一套长期可查阅、可扩展的学习资料。学完后，遇到 three.js 的常见能力点时，能够在本工作区的学习路线、资源和课程中找到对应教程或入口。

## 成功标准
- 能用 Vite 和原生 JavaScript 搭建 three.js 项目，并解释 Renderer、Scene、Camera、Object3D、Geometry、Material、Light、Texture、Loader、Controls 的关系。
- 能独立完成常见实践：第一幅画面、响应式渲染、相机控制、光照材质、纹理、动画循环、模型加载、调试与性能排查。
- 能把 three.js 能力整理成系统地图，提到某个能力时，可以回到 `README.md`、`CHEATSHEET.md`、`RESOURCES.md` 或 `cheatsheets/` 找到学习入口。
- 能在后续长期学习中扩展到进阶主题，例如后处理、着色器、物理/交互、模型资产管线、WebGPU、Gaussian Splatting 或框架集成。

## 当前水平
用户具备较好的 JavaScript 基础，了解过 Canvas。需要从 three.js 本身的工程搭建、核心对象模型、3D 渲染概念和实践项目开始建立系统认知。

## 约束
- 教程正文、标题、说明、示例界面文案和代码注释使用中文；路径、命令、包名、API 名称和代码标识符可以保留英文。
- 可以接受英文资料作为参考来源，尤其是官方文档、一手资料和高质量社区内容。
- 学习方式采用“概念系统讲清楚 + 边做项目边学”。
- 学习节奏面向长期系统学习，不追求一天内覆盖所有能力。
- 示例默认使用原生 JavaScript、ES Modules、Vite 和浏览器开发工具。
- 默认学习氛围：从 0 快速上手并建立系统理解；实战和理论结合；能写示例就尽量提供覆盖关键变体的范例；课程可作为速查表 / 参考手册使用。

## 暂不纳入
- 暂不把 React Three Fiber、Vue 集成、Babylon.js、Unity/Unreal、完整游戏引擎开发作为入门主线。
- 暂不在第一阶段深入 WebGPU、底层 WebGL 着色器、复杂物理引擎、Gaussian Splatting 原理或大型 3D 资产制作流程。
